# Allelix Example Reports

Pre-generated sample reports demonstrating Allelix output. The reports here
let you see exactly what `allelix analyze` produces without needing your own
genotype file or downloaded databases.

## Quick Start

Open `sample_reports/demo_report.html` in any browser to view the rendered
report.

To regenerate the reports yourself from the included demo input:

```bash
./regenerate.sh
```

Requires `allelix` installed and `allelix db update` already run.

To generate a report from your own genotype file:

```bash
allelix db update
allelix analyze YOUR_FILE.txt --output report.html
```

## What's Here

```
examples/
├── README.md
├── regenerate.sh                    # Reproduces sample_reports/ from sample_input/
├── sample_input/
│   └── demo_23andme.txt             # 10,000-variant subset (~244 KB)
└── sample_reports/
    ├── demo_report.html             # Self-contained HTML report
    ├── demo_report.json             # Machine-readable JSON
    └── demo_stats.txt               # Stats output
```

### `sample_input/demo_23andme.txt`

A 10,000-variant subset of a publicly donated genotype from openSNP user
1190. The subset preserves every rsID that produced a database annotation
(ClinVar, PharmGKB, or GWAS Catalog), then pads with random variants from
the source file to a realistic chip-scale per-chromosome distribution.

The file is structurally identical to a full 23andMe export — same delimiters,
same column order, same header. Parsing it produces a normal report; it
just has fewer variants than a real chip.

Generated by `scripts/subset_for_demo.py` (in the repo root). With the
same input and the same pinned random seed, the script reproduces the
same subset.

### `sample_reports/demo_report.html`

Self-contained HTML report. Inline CSS, no external dependencies, opens
in any browser. Shows 358 annotations across ClinVar (239), PharmGKB (26),
and GWAS Catalog (93) — including pathogenic clinical variants, drug
response associations, and well-replicated genetic associations.

### `sample_reports/demo_report.json`

Machine-readable JSON with the same data. Useful for piping into other
tools or building custom report formats.

### `sample_reports/demo_stats.txt`

Output of `allelix stats` — total variants, no-call rate, heterozygosity,
per-chromosome counts.

## Regenerating the input subset

If you want to rebuild `sample_input/demo_23andme.txt` from a new source
file (e.g., a more recent openSNP donation, or a different chip version),
run `scripts/subset_for_demo.py`:

```bash
python3 scripts/subset_for_demo.py \
    --input PATH-TO-FULL-23ANDME-FILE \
    --databases PATH-TO-ALLELIX-DB-CACHE \
    --output examples/sample_input/demo_23andme.txt \
    --target-variants 10000 \
    --exclude-snpedia
```

Then run `./regenerate.sh` to refresh `sample_reports/`. Commit both
together.

## What this does NOT contain

- No SNPedia data, in any form. SNPedia's CC BY-NC-SA license is
  incompatible with redistribution in this repository.
- No content derived from any private, identifying, or
  consent-restricted dataset. The donor genome used for the subset
  (openSNP user1190) was voluntarily released into the public domain
  under CC0.
- No telemetry, tracking, or external resource loads. The HTML report
  is fully self-contained.
